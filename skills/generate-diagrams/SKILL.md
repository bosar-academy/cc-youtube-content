# Whiteboard Diagram Generator

Generate professional whiteboard-style diagrams for YouTube videos. Each diagram is a **single cohesive illustration** generated by Gemini — NOT assembled from Excalidraw primitives. This produces visually unified diagrams where characters, icons, cards, arrows, and text all share the same hand-drawn art style.

Output: `.excalidraw` files (with the illustration embedded as an image) and standalone `.png` files.

## The Core Approach

**OLD approach (DON'T DO THIS):** Build diagrams from Excalidraw rectangles + embedded Gemini icon PNGs + fetched logos. This creates visual inconsistency — Excalidraw text looks different from illustrated text, embedded PNGs look foreign next to Excalidraw shapes.

**NEW approach:** Generate the **entire diagram as one image** via Gemini, using reference screenshots as style examples. One prompt → one cohesive illustration. Then embed it in an Excalidraw file for editability.

## Style Guide (embed in EVERY Gemini prompt)

Include this style block in every generation prompt:

```
STYLE REQUIREMENTS (match the provided example screenshots exactly):
- Hand-drawn whiteboard sketch aesthetic — like the best YouTube whiteboard explainers
- Soft pastel color fills for cards/containers: cream/beige, light blue, soft green, light pink
- ALL text is hand-drawn/illustrated (part of the image) — dark text on light backgrounds
- Small delicate sketch icons (key, clipboard, link, clock, browser) drawn in the same style as everything else
- Simple cartoon characters (stick-figure-like, whiteboard doodle style) when showing people
- Real brand logos/icons drawn in recognizable but sketch-consistent style (YouTube play button, LinkedIn "in", X logo, Instagram gradient square)
- Thin pencil-like lines for arrows and connections — NOT thick bold strokes
- Generous whitespace — clean and spacious, NOT crowded
- Numbered circle badges for steps (hand-drawn circles with numbers inside)
- Floating annotation text with thin arrows — like handwritten margin notes
- White/off-white background
- NO bold saturated color blocks — everything is soft and pastel
- NO digital/polished look — it should feel hand-drawn on a whiteboard
- Size: wide format, 1920x1080 pixels (16:9), suitable for YouTube video overlay
```

## Diagram Types & Prompt Templates

### Type 1: Chaos / Before-After
*Reference: Screenshot 1 — "The Manual Way"*

```
Generate a hand-drawn whiteboard diagram showing [THE PROBLEM/CHAOS].

Layout:
- Left side: [SOURCE/INPUT] drawn as a large recognizable icon
- Center: A mess of tangled, criss-crossing arrows going in all directions (conveying chaos/complexity)
- Right side: Multiple destination items stacked vertically, each with:
  - Its recognizable brand icon (drawn in sketch style)
  - A small clock/timer icon next to it
  - A "?" mark suggesting uncertainty
- Bottom-left: A simple cartoon character sitting at a laptop, looking overwhelmed
- Top: Title text "[TITLE]" in hand-drawn lettering with a red "X" or cross mark
- Bottom: Annotation text "[TIME ESTIMATE]" in smaller hand-drawn text

[PASTE STYLE REQUIREMENTS BLOCK]
```

### Type 2: Card Columns
*Reference: Screenshot 2 — "What Blotato Can Do"*

```
Generate a hand-drawn whiteboard diagram showing [FEATURE/CAPABILITY OVERVIEW].

Layout:
- Top: Large title text "[TITLE]" in bold hand-drawn lettering

```

*Reference: Screenshot 3 — "Setup Steps"*

Generate a hand-drawn whiteboard diagram showing [SETUP/STEPS PROCESS].

Layout:

  - Step 1: Circle badge with "1", checkbox square, large text "[STEP NAME]", small sketch icon on the right (e.g., key icon)
  - Step 2: Circle badge with "2", checkbox square, large text "[STEP NAME]", small sketch icon (e.g., browser window)
  - Step 3: Circle badge with "3", checkbox square, large text "[STEP NAME]", brand logos on the right ([PLATFORM ICONS])

[PASTE STYLE REQUIREMENTS BLOCK]
```

### Type 4: Vertical Flow with Branches
*Reference: Screenshot 4 — "Flow with Branches"*

```

Layout:
- Main vertical flow (left side), connected by downward arrows:

  - Branch cards with pastel fills (light blue, light pink, soft green)
  - Each card: Platform/output name in bold + italic sub-label describing the adaptation
  - Thin arrows connecting main flow to branch cards
- Overall flow: top to bottom with branches spreading right

[PASTE STYLE REQUIREMENTS BLOCK]

## Process

### Step 1: Extract diagram requests from script

Read the script and find all `[DIAGRAM: name]` markers. For each one, read the surrounding context to understand what the diagram should communicate.

### Step 2: Design each diagram

For each diagram:
1. **Pick the closest diagram type** from the templates above
2. **Read the matching reference screenshot** from `~/Projects/content/diagrams/references/`
3. **Fill in the template** with specific content from the script
4. **List which reference screenshots to pass as `--examples`** (always include the closest match + 1-2 others for general style)

### Step 3: Select reference examples

Pick 2-3 reference screenshots that best match each diagram's type:

| Diagram Type | Primary Example | Secondary Examples |
|-------------|----------------|-------------------|
| Chaos / Before-After | Screenshot 1 (Manual Way) | Screenshot 4 (Flow) |
| Card Columns | Screenshot 2 (Blotato) | Screenshot 5 (Recipe) |
| Numbered Steps | Screenshot 3 (Setup) | Screenshot 4 (Flow) |
| Flow with Branches | Screenshot 4 (Flow) | Screenshot 3 (Setup) |
| Recipe / Concept | Screenshot 5 (Recipe) | Screenshot 2 (Blotato) |
| Architecture | Screenshot 4 (Flow) | Screenshot 2 (Blotato) |

### Step 4: Generate via Gemini

```bash
export $(grep GEMINI_API_KEY ~/Projects/content/my-thumbnails/.env | xargs)

REFS=~/Projects/content/diagrams/references
GEN=~/Projects/content/my-thumbnails/.claude/skills/youtube-thumbnail/scripts/generate_thumbnail.py

# Generate full-scene diagram (pick 2-3 matching examples)
python3 "$GEN" \
  --headshot ~/Projects/content/assets/headshots/bohdan-cutout.png \
  --examples "$REFS/Screenshot 2026-03-25 at 10.22.41 AM.png" "$REFS/Screenshot 2026-03-25 at 10.22.46 AM.png" "$REFS/Screenshot 2026-03-25 at 10.22.51 AM.png" \
  --prompt "[FULL DIAGRAM PROMPT FROM TEMPLATE — include ALL content, layout description, AND style requirements block]" \
  --output /tmp/diagram-assets/[diagram-name].png \
  --no-style
```

**CRITICAL RULES:**
- Always use `--no-style` (we provide our own style in the prompt)
- Always pass `--examples` with 2-3 reference screenshots
- The prompt must describe the ENTIRE diagram: layout, content, text, icons, characters, colors, annotations
- Include the full STYLE REQUIREMENTS block in every prompt
- Be VERY specific about spatial layout (left side, right side, top, bottom, center)
- Specify exact text content for all labels (Gemini will render it as hand-drawn text)
- Save outputs to `/tmp/diagram-assets/`

### Step 5: Verify output dimensions

```bash
python3 -c "
from PIL import Image
import os
for f in sorted(os.listdir('/tmp/diagram-assets')):
    if f.endswith('.png'):
        img = Image.open(f'/tmp/diagram-assets/{f}')
        w, h = img.size
        print(f'{f}: {w}x{h} ratio={w/h:.2f}')
"
```

Gemini should output approximately 16:9 images. If not, note the actual dimensions for embedding.

### Step 6: Embed into Excalidraw (optional — for editability)

To create `.excalidraw` files that can be opened and edited:

```bash
# Create a minimal spec that just embeds the full-scene image
cat > /tmp/diagram-specs/[name].json << 'SPECEOF'
{
  "titles": [],
  "nodes": [],
  "arrows": [],
  "images": [
    {"src": "diagram", "x": 0, "y": 0, "w": 1920, "h": 1080}
  ]
}
SPECEOF

python3 ~/Projects/content/scripts/_tools/generate_diagrams.py build \
  --spec /tmp/diagram-specs/[name].json \
  --images "diagram:/tmp/diagram-assets/[name].png" \
  --output ~/Projects/content/assets/diagrams/[video-slug]/[name].excalidraw
```

**Adjust `w` and `h`** to match actual image dimensions from Step 5.

For combined file with multiple diagrams:
```bash
python3 ~/Projects/content/scripts/_tools/generate_diagrams.py combine \
  --specs /tmp/diagram-specs/1-*.json /tmp/diagram-specs/2-*.json \
  --images "diagram1:/tmp/diagram-assets/diagram1.png" "diagram2:/tmp/diagram-assets/diagram2.png" \
  --output ~/Projects/content/assets/diagrams/[video-slug]/all-diagrams.excalidraw \
  --cols 2 --gap 100
```

### Step 7: QA — Compare to references

For each generated diagram:
1. **Open the PNG** and visually compare to the reference screenshots
2. Check: Does it have the same hand-drawn whiteboard feel?
3. Check: Are colors soft/pastel (not bold/saturated)?
4. Check: Is text readable and hand-drawn looking?
5. Check: Are icons/characters integrated (same art style as everything else)?
6. Check: Is there generous whitespace?

**If the output doesn't match the reference quality:**
- Refine the prompt with more specific layout instructions
- Add more `--examples` (pass all 5 reference screenshots)
- Use `--reference` to pass the previous attempt and ask Gemini to improve it:

```bash
python3 "$GEN" \
  --headshot ~/Projects/content/assets/headshots/bohdan-cutout.png \
  --examples "$REFS/Screenshot 2026-03-25 at 10.22.46 AM.png" \
  --reference /tmp/diagram-assets/[previous-attempt].png \
  --prompt "Improve this diagram to better match the example style. Specifically: [WHAT TO FIX]. Keep the same content and layout but make it more hand-drawn, more whiteboard-like, with softer colors." \
  --output /tmp/diagram-assets/[name]-v2.png \
  --no-style
```

## Prompt Writing Tips

### Be spatially explicit
BAD: "Show a flow from input to output"
GOOD: "Left side: YouTube play button icon (large, red). Center: 5 tangled arrows crossing each other going in all directions. Right side: 4 platform icons stacked vertically (LinkedIn, X, Instagram, Facebook) each with a small clock and question mark next to them."

### Specify all text content
BAD: "Add labels to each step"
GOOD: "Step 1 text reads 'Get API key'. Step 2 text reads 'Add MCP config'. Step 3 text reads 'Connect accounts'."

### Describe the feel, not just the content
BAD: "Three columns of features"
GOOD: "Three slightly overlapping cards — the left one has a light blue paper-like fill, the middle one cream/beige, the right one soft green. Each feels like a hand-written index card. The overlap gives a casual, laid-back feel like cards spread on a desk."

### Use character descriptions
BAD: "Show a person"
GOOD: "Bottom-left: a simple cartoon character (stick-figure style, like a whiteboard doodle) sitting at a laptop, shoulders slightly hunched, conveying 'overwhelmed'. Not detailed — just enough lines to read the emotion."

## Error Handling

| Error | Fix |
|-------|-----|
| Gemini returns no image | Simplify prompt, reduce detail, retry |
| Image is too small | Specify "Size: 1920x1080 pixels" explicitly in prompt |
| Text is unreadable | Add "All text must be large and clearly readable at 1080p" to prompt |
| Style doesn't match references | Add more --examples, be more specific about pastel colors and hand-drawn feel |
| Brand logos look wrong | Describe them explicitly: "YouTube red play button triangle", "LinkedIn blue square with white 'in'" |
| Colors too saturated/bold | Add "SOFT pastel fills only — cream, light blue, soft green — NOT bold colors" to prompt |
| Too crowded | Add "generous whitespace, at least 30% of the image should be empty white space" |
